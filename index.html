Toutes les commandes vues dans tes TPs, classées par catégorie

SELECT Sélectionner des données

SELECT * FROM actor;

SELECT first_name, last_name FROM actor;

-- Évite les doublons
SELECT DISTINCT last_name FROM actor;

WHERE Filtrer les lignes

SELECT * FROM actor
WHERE first_name = 'SCARLETT';

-- Opérateurs: = != < > <= >= LIKE IN
WHERE last_name LIKE '%SON';
WHERE last_name = 'JOHANSSON';

Agrégats COUNT, AVG, SUM…

-- Compter les acteurs
SELECT COUNT(*) AS nombre_acteurs
FROM actor;

-- Longueur moyenne des films
SELECT AVG(length) FROM film;
-- Résultat : 115.2720

GROUP BY Regrouper

-- Noms de famille en doublon
SELECT last_name, COUNT(*) AS nb
FROM actor
GROUP BY last_name
HAVING COUNT(*) > 1;

JOIN Jointures — le cœur de tes TPs

Sakila (acteurs ↔ films)

SELECT a.first_name, a.last_name, f.title
FROM actor a
JOIN film_actor fa ON a.actor_id = fa.actor_id
JOIN film f ON fa.film_id = f.film_id;

Sakila (films ↔ catégorie)

SELECT c.name, AVG(f.length)
FROM film f
JOIN film_category fc ON f.film_id = fc.film_id
JOIN category c ON fc.category_id = c.category_id
GROUP BY c.name;

📐 Structure d'une requête SQL

L'ordre des clauses est OBLIGATOIRE — impossible de mettre HAVING avant GROUP BY !
OrdreClauseRôleExemple
1SELECTColonnes à afficherSELECT name, COUNT(*)
2FROMTable sourceFROM actor
3JOIN…ONLiaison entre tablesJOIN film ON …
4WHEREFiltre AVANT groupementWHERE name = 'X'
5GROUP BYRegrouper les lignesGROUP BY last_name
6HAVINGFiltre APRÈS groupementHAVING COUNT(*) > 1
7ORDER BYTrier les résultatsORDER BY nb DESC

⚠️ Pièges classiques

WHERE vs HAVING

WHERE filtre les lignes AVANT le GROUP BY
HAVING filtre les groupes APRÈS le GROUP BY

❌ WHERE COUNT(*) > 1 → Erreur !
✅ HAVING COUNT(*) > 1 → Correct
JOIN vs LEFT JOIN

JOIN = ne retourne que les lignes qui ont une correspondance dans les deux tables

LEFT JOIN = retourne TOUTES les lignes de la table gauche, même sans correspondance (NULL)
